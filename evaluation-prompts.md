## Feature 迭代（20 题）

### F01 [Feature迭代]

商品详情页目前只展示主图，后端 Product 模型已有 `images` JSON 字段但前端未使用。请为商品详情页增加多图轮播展示能力：支持左右切换、缩略图导航、点击放大预览；若商品只有一张图则隐藏轮播控件；轮播区域需与现有价格、库存、加购按钮布局协调，移动端也要正常滑动。完成后我能在任意商品详情页看到多图切换效果。

### F02 [Feature迭代]

订单模型已定义 `shipped` 和 `completed` 状态，但后端缺少对应接口，前端订单列表和详情页也无法操作。请补全订单履约链路：已支付订单支持「模拟发货」，已发货订单支持「确认收货」变为已完成；订单列表的状态筛选和详情页的状态标签、操作按钮要同步更新，状态流转要有合理的权限校验（只能操作自己的订单）。我能在我的订单页走完 pending → paid → shipped → completed 全流程。

### F03 [Feature迭代]

个人中心 Profile 页目前只能查看昵称、邮箱、手机号，无法修改。请增加资料编辑能力：支持修改昵称和手机号，头像支持本地上传并在 Header 下拉菜单同步展示；表单需做前端校验（手机号格式、昵称长度），保存成功后页面即时刷新展示新资料。修改后我在 Profile 页和顶部导航都能看到更新后的信息。

### F04 [Feature迭代]

收货地址表单里省、市、区目前是三个独立文本框，填写体验差且容易出错。请把地址新增/编辑表单改为省市区三级联动选择器，选完省再加载市、选完市再加载区，支持常见省份数据；编辑已有地址时要正确回显三级选项；结算页内嵌的新增地址弹窗也要一并改造。我在地址管理页和结算页都能用级联选择器完成地址填写。

### F05 [Feature迭代]

顶部搜索框目前提交后直接跳转商品列表，没有搜索历史也没有热门词引导。请扩展搜索体验：用户搜索过的关键词保存在本地，再次聚焦搜索框时展示最近 10 条历史记录并支持逐条删除和一键清空；同时在搜索框下方展示 5 个热门搜索词（可写死种子数据），点击热门词直接触发搜索。这些交互在首页和商品列表页顶栏都要生效。

### F06 [Feature迭代]

用户浏览商品时没有收藏能力。请为商品详情页和商品列表卡片增加收藏/取消收藏功能，收藏状态需登录后持久化到后端；个人中心增加「我的收藏」入口，展示收藏商品列表，支持取消收藏和点击跳转详情。未登录用户点击收藏应引导登录。我登录后收藏几件商品，能在个人中心看到完整列表。

### F07 [Feature迭代]

购物车目前只能全量结算，无法勾选部分商品下单。请给购物车列表增加多选能力：每行有复选框，顶部有全选/反选，底部结算栏实时显示已选商品数量和合计金额，「去结算」只携带选中商品进入确认订单页；若未选中任何商品则结算按钮置灰并提示。后端创建订单接口需支持传入指定 cart_item id 列表而非整辆购物车。

### F08 [Feature迭代]

结算页目前只有商品总额，没有优惠能力。请引入优惠券功能的基础闭环：后端新增用户优惠券表，种子数据放几张满减券；结算页展示可用/不可用优惠券列表并说明不可用原因（未达门槛、已过期等），用户选一张券后实时重新计算应付金额；下单时将券信息写入订单。我在结算页选券后能看到折扣后的最终价格并完成下单。

### F09 [Feature迭代]

已完成的订单目前无法评价商品。请增加商品评价能力：用户在订单详情页对已完成的订单可以逐商品提交评分（1-5 星）和文字评价；商品详情页底部展示该商品的历史评价列表（用户名脱敏、评分、内容、时间），支持分页；同一订单同一商品只能评价一次。我完成一笔订单后在订单详情提交评价，再回到商品详情页能看到刚写的评价。

### F10 [Feature迭代]

确认订单页的备注输入框是空白文本域，用户不知道该写什么。请优化备注体验：提供「工作日配送」「周末配送」「放门口」「需要电话联系」等快捷标签，点击标签追加到备注框，标签可多选、可取消，备注框仍支持自由输入；已选标签在提交订单时一并保存到订单 remark 字段，订单详情页要展示完整备注内容。

### F11 [Feature迭代]

首页「精选推荐」目前是固定商品，不会变化。请改为基于用户行为的个性化推荐：记录用户最近浏览过的商品分类（存 localStorage 即可），登录用户合并后端购物车/订单中的分类偏好；首页推荐区优先展示同分类的其他商品，冷启动时回退到销量排序；模块标题根据是否有偏好数据显示「为你推荐」或「热门精选」。我浏览几个数码类商品后刷新首页，应看到推荐区偏向数码分类。

### F12 [Feature迭代]

用户选购时缺少对比工具。请在商品列表页和详情页增加「加入对比」按钮，最多同时对比 3 个商品；页面底部或侧边浮出对比栏，显示已选商品缩略图，点击「开始对比」进入对比页，以表格形式并列展示价格、库存、销量、分类、描述摘要等字段，支持移除单个对比项。对比状态在页面跳转间保持，直到用户清空。

### F13 [Feature迭代]

已发货订单在详情页只有状态文字，看不到物流进展。请为已发货及之后状态的订单增加模拟物流时间轴：展示「已揽收→运输中→派送中→已签收」等节点及 fictive 时间，当前节点高亮；pending/paid 状态不显示物流模块；shipped 状态显示前两个节点完成，completed 显示全部完成。我在不同状态的订单详情页能看到对应的物流轨迹 UI。

### F14 [Feature迭代]

购物车里的商品如果被下架或库存不足，用户往往下单时才发现。请增加购物车健康检查：进入购物车页时校验每个 cart_item 对应商品的 status 和 stock，库存不足的行标红并显示「仅剩 X 件」，已下架商品标记「已失效」且数量不可调、不可被选中结算，顶部 banner 汇总提示有多少件商品需要处理，提供「一键清除失效商品」按钮。后端 cart 接口返回数据中需包含足够的商品状态字段。

### F15 [Feature迭代]

商品详情页目前不展示价格变动信息。请增加价格变动提示：后端记录商品最近 30 天的 price 变更历史（新表或 JSON 字段），详情页若当前价格低于 30 天内最高价则展示「近 30 天降价 X 元」绿色标签；若当前价就是最低价则展示「历史低价」标签；无变动则不显示。标签放在价格区域旁边，不影响现有折扣徽章逻辑。

### F16 [Feature迭代]

购物车删除商品目前只能逐条点删除且没有批量操作。请增加批量管理能力：每行前面加复选框，顶部提供全选和「批量删除」按钮，删除前弹出 ConfirmModal 确认；批量删除后刷新列表和 Header 角标；若购物车已有其他勾选交互，复选框状态应统一管理，避免出现两套互相独立的 checkbox。

### F17 [Feature迭代]

注册流程目前只有用户名邮箱密码，缺少手机验证环节。请扩展注册页：增加手机号输入框和「获取验证码」按钮，验证码 60 秒倒计时、6 位数字；后端新增发送验证码和校验接口（开发环境直接在响应里返回验证码即可），注册提交时一并校验；已有账号的手机号不可重复注册。我能走通填手机号→收验证码→完成注册的全流程。

### F18 [Feature迭代]

订单详情页信息完整但不方便留存。请增加「打印订单」功能：详情页顶部加打印按钮，点击后打开适合 A4 打印的预览视图，包含订单号、下单时间、商品明细表格、收货地址、支付方式、金额合计，隐藏导航栏和无关按钮；使用 CSS `@media print` 或直接新窗口渲染均可，打印预览中布局不能错乱。

### F19 [Feature迭代]

商品列表页目前只有网格卡片视图。请增加列表/网格视图切换：工具栏加 toggle 按钮，网格视图保持现有卡片样式，列表视图改为横向排列（左图右文，展示名称、价格、销量、分类）；用户选择的视图模式存入 localStorage，下次进入页面恢复。两种视图下筛选、排序、分页功能均正常工作。

### F20 [Feature迭代]

新增收货地址时手动填写省市区很繁琐。请在地址表单上方增加「智能识别」输入框：用户粘贴一段完整地址文本（如「张三 13800138000 北京市朝阳区建国路88号」），点击识别按钮后自动解析填充收货人、手机号、省市区和详细地址；解析失败时给出提示但不覆盖已填内容。识别逻辑可前端实现简单规则，结算页弹窗里同样可用。

## 0-1 构建（20 题）

### O01 [0-1]

消息通知中心：实现站内消息体系，涵盖订单支付成功、发货提醒、售后进度变更等事件触发的自动通知写入；Header 区域放置铃铛入口并展示未读角标，点击进入 `/notifications` 消息列表页，支持按类型筛选、单条标记已读、一键全部已读，消息卡片可跳转至关联订单或业务页面；未登录状态下隐藏入口，已读与未读状态在列表中有明显视觉区分。

### O02 [0-1]

限时秒杀：实现完整的秒杀活动能力，活动配置包含关联商品、秒杀专属价格、起止时间与独立秒杀库存；首页设置秒杀入口区块，独立页面 `/flash-sale` 分区展示「正在进行」与「即将开始」的活动场次，场次卡片附带实时倒计时与库存余量；加购链路需校验活动有效期与秒杀库存，秒杀价在商品卡片与详情页中以醒目样式呈现，库存耗尽或活动结束后禁止下单。

### O03 [0-1]

品牌专区：实现品牌维度的商品聚合展示，维护品牌名称、标识、简介故事等基础信息，商品与品牌建立关联关系；品牌馆列表页 `/brands` 以卡片形式陈列各品牌，品牌详情页 `/brand/:slug` 展示品牌介绍及旗下商品网格并支持分页浏览；商品详情页展示所属品牌名称并提供跳转链接，整体视觉风格与商城主站保持一致。

### O04 [0-1]

商品问答：实现「问大家」互动能力，用户可在商品详情页底部切换至问答 Tab，浏览历史提问与官方回复并支持分页加载；登录用户可提交文字问题，问题经后端持久化后在列表中展示提问者与时间信息，商家或系统回复后问题状态更新为已回答并在原问题下方展示回复内容；问答模块与商品详情页布局自然融合，空状态下给出引导文案。

### O05 [0-1]

积分商城：实现用户积分账户与兑换体系，用户注册时获得初始积分，订单完成后按实付金额比例累积积分并写入流水明细；独立页面 `/points-mall` 陈列可兑换的优惠券、虚拟权益等兑换项，标注所需积分与库存限制；个人中心展示当前积分余额与近期流水记录，兑换操作扣减积分并生成兑换记录，积分不足或库存耗尽时给出明确提示。

### O06 [0-1]

售后退换货：实现订单完成后的售后申请全流程，支持针对订单内单个商品发起退货或换货申请，填写原因描述并提交；售后单具备 pending、approved、rejected、completed 等状态流转，用户在 `/refunds` 列表页查看全部售后记录及当前进度，详情页展示申请信息与处理结果；已完成订单详情页提供「申请售后」入口，跳转至 `/refund/apply` 表单页完成选择与提交。

### O07 [0-1]

浏览足迹：实现商品浏览行为的记录与回溯，用户进入商品详情页时自动记录浏览条目并按时间倒序维护，本地保留最近 50 条且同商品去重，登录用户的记录同步至服务端；独立页面 `/history` 按「今天」「昨天」「更早」分组展示浏览过的商品卡片，支持逐条移除与一键清空，卡片点击可回到对应商品详情，空状态给出逛一逛的引导。

### O08 [0-1]

每日签到：实现连续签到奖励机制，签到规则支持按连续天数递增发放积分或优惠券，中断后连续天数重置；个人中心提供签到入口，独立页面 `/check-in` 以月历形式展示当月签到情况，已签日期做标记，页面顶部展示连续签到天数与今日奖励说明；用户每日仅可签到一次，签到成功后即时更新日历状态并发放对应奖励。

### O09 [0-1]

发票管理：实现电子发票申请与管理能力，已支付或已完成的订单详情页提供「申请发票」入口，弹窗表单支持选择个人或企业抬头、填写抬头名称与税号等信息；发票申请记录汇总至 `/invoices` 列表页，展示关联订单、申请金额与处理状态（待开具、已开具）；同一订单限制申请一次，企业抬头必填税号，表单提交前做字段校验。

### O10 [0-1]

帮助中心：实现面向用户的问题自助查阅体系，文章内容支持分类归属、Markdown 渲染、排序权重与浏览量统计；帮助首页 `/help` 采用左侧分类导航、右侧文章列表的双栏布局，文章详情页 `/help/:id` 渲染正文并自动生成文内目录锚点；页面底部提供「这篇有帮助吗」反馈交互用于统计有用计数，帮助入口可从页脚链接进入，分类与文章数据通过种子脚本预置常见问题内容。

### O11 [0-1]

拼团购买：实现社交拼团下单能力，活动配置包含拼团专属价格、成团所需人数与拼团有效时长；商品详情页展示拼团价与「发起拼团」操作，团长发起后生成分享链接 `/group/:id`，其他用户通过链接加入拼团；拼团页面实时展示当前参团人数、还差几人成团及剩余时间，人数凑齐后拼团成功并生成订单，超时未成团则自动失效并释放库存占用。

### O12 [0-1]

礼品卡：实现礼品卡的购买、绑定与消费抵扣链路，提供固定面额（如 100、200、500 元）的礼品卡购买流程并完成模拟支付，购买成功后生成唯一卡号；用户可在 `/gift-card` 页面通过 Tab 切换「购买礼品卡」与「绑定礼品卡」，绑定后余额展示于个人中心；结算页支持选择已绑定的礼品卡抵扣订单金额，抵扣后展示剩余应付金额，余额不足时按实际余额部分抵扣。

### O13 [0-1]

到货通知：实现缺货商品的到货订阅能力，商品库存为零时在详情页展示「到货通知我」按钮，登录用户点击后写入订阅记录；当商品库存由零恢复为正数时，系统自动向已订阅用户推送到货提醒消息；订阅状态在商品详情页有明确反馈（已订阅 / 取消订阅），到货提醒可出现在消息通知中心或个人中心的提醒列表中，点击后跳转至对应商品详情页。

### O14 [0-1]

会员等级体系：实现基于累计消费的会员成长体系，配置普通、银牌、金牌、钻石等等级及其消费门槛与专属折扣率；用户累计消费金额在订单完成后自动累加，个人中心 Profile 页展示当前等级徽章、升级进度条及各级权益说明；商品详情页与结算页对会员用户展示专属折扣价，价格区域同时呈现原价与会员价，让用户感知等级带来的实际优惠。

### O15 [0-1]

客服工单：实现用户与客服之间的工单沟通渠道，用户可在 `/support` 页面提交工单，选择问题分类、填写主题与详细描述；工单列表页 `/support/tickets` 展示全部历史工单及 open、replied、closed 等状态，详情页呈现用户原始描述与客服回复内容的时间线；工单提交成功后给出确认反馈，回复内容支持富文本或分段展示，方便用户追踪问题处理进展。

### O16 [0-1]
